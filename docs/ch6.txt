CHAPTER 6

DISCUSSION

6.1 Overview

This chapter interprets the results from Scenario 1 and connects them to the fairness theorems in Chapter 4. The aim is to explain what the results show and why they matter. No new proofs are introduced.

Scenario 1 produces a deterministic execution trace sigma from the initial state x0. The trace is generated by repeated application of the transition function delta. When the execution reaches Phase P7, stuttering applies. The final state is repeated, and the trace becomes infinite.

Each position in the trace corresponds to a state of the protocol. Atomic predicates are evaluated at each position. This connects the operational model in Chapter 3 with the LTL semantics in Chapter 4 and allows the fairness properties to be evaluated along the execution.

6.2 Interpretation of Priority Fairness

The results show that priority fairness holds at every matching step. At each occurrence of Phase P3, the mechanism selects the highest-priority feasible buyer and seller. No higher-priority order is skipped.

This confirms that the greedy matching rule is correctly enforced by the transition system. The selection process is consistent across all matching steps. As a result, the execution avoids arbitrary decisions.

This property is specific to the matching phase. Outside Phase P3, no selection takes place. The protocol only updates state information or progresses through phases. This separation clarifies where priority constraints apply.

6.3 Interpretation of Quantity Fairness

The results show that quantity is preserved at every step of the execution. For each order, the initial quantity is equal to the sum of allocated and residual quantities.

This confirms that the match record provides a correct accounting structure. Each matching step updates the allocation, and the residual quantity reflects what remains. The relation between these values is maintained throughout the execution.

After the terminal phase, stuttering preserves this relation. Since the state no longer changes, the same accounting identity holds at all later positions in the trace.

This ensures that the protocol maintains internal consistency. All traded units are recorded, and no quantity is lost or created.

6.4 Interpretation of Price Fairness

The results show that the clearing price satisfies the pricing rule whenever it is defined. The price lies within the marginal bid and ask bounds and follows the deterministic rule.

At the start of the execution, no clearing price is defined because no matches exist. The condition is therefore inactive. Once a marginal pair is determined, the clearing price becomes defined. From that point onward, the pricing rule is satisfied at each step.

This confirms that the pricing mechanism is consistent with the outcome of the matching phase. The clearing price reflects the structure of the match record and remains stable after termination.

6.5 Implications

These results show that fairness can be expressed as invariants over execution traces. Each property holds at every position of the trace, as stated formally by Theorems 1–3 in Chapter 4.

The use of LTL provides a uniform way to state these properties. The form G phi ensures that correctness is preserved throughout the execution.

Stuttering allows finite executions to be treated as infinite traces. This makes the execution compatible with LTL semantics.

Lifting connects state-level predicates to their evaluation along the trace. This allows the operational definitions in Chapter 3 to be used directly in the temporal framework of Chapter 4.

Together, these elements provide a clear method for verifying fairness in the protocol.

6.6 Limitations

This chapter analyzes one concrete scenario. It illustrates how the properties appear in an execution but does not establish general correctness. The general results are proven in Chapter 4.

The current work focuses on three fairness properties. Other properties, such as maximality and justified rejection, are not included in the verified core.

The temporal specification uses invariants of the form G phi. It does not yet address properties that require other temporal operators.

6.7 Recommendations for Future Work

Future work can extend the set of verified properties. Additional fairness conditions can be formalized using the same framework.

Temporal properties beyond invariants can also be studied. Examples include termination and phase progression. These require other LTL operators.

More scenarios can be analyzed to illustrate different execution patterns. This can provide further validation of the model.

The framework can also be used as a reference specification for implementations. System executions can be compared with the verified model to check correctness.
