CHAPTER 7
CONCLUSION

This thesis presents a precise and machine-checked verification of three fairness properties in the MUDA protocol. The protocol is modeled as a deterministic state-transition system with seven phases, and its behavior is analyzed over complete execution traces using Linear Temporal Logic. Each execution is treated as a full sequence of states, extended to an infinite trace by repeating the terminal state after Phase P7.

The main contribution is not just showing that MUDA works in selected examples. It shows that the protocol satisfies the verified fairness properties for every execution trace generated by iterating the deterministic transition function from an initial state of the form initial_state bs as_list, for arbitrary input lists bs and as_list. This removes dependence on testing and simulation, which can only cover a limited number of cases. Instead, the results establish that these properties hold for all such executions.

The work also clarifies what fairness means in the context of MUDA. Rather than checking properties only at the end of the auction, the thesis shows invariants over the execution trace. Priority is respected when matching decisions are made, quantities satisfy a conservation-style accounting identity, and whenever a clearing price is defined by determine_clearing_price from the match record it follows the deterministic rule and satisfies the marginal bounds. Moreover, once the protocol has moved past pricing (Phase P4), if the state stores a value in the clearing_price field then it agrees with determine_clearing_price for that state.

Another important outcome is the use of a machine-checked proof system. All results are verified in Rocq, and no part of the proof is left incomplete. This removes hidden errors in the reasoning. It also provides a concrete reference that others can inspect, reuse, or extend.

The modeling approach also strengthens the connection between theory and implementation. The same definitions used in the proofs also describe executable behavior. The example scenario shows how the protocol evolves step by step, making the formal model easier to understand and relate to real systems.

Overall, the impact of this work is that it provides a formally specified and machine-verified model of the auction protocol. It gives guarantees about fairness that hold for all execution traces generated by the model from initial_state, not just observed cases. This level of assurance is important in domains where incorrect behavior can lead to financial loss or unfair outcomes.

Future work

The current development verifies three fairness properties: priority, quantity, and price fairness. Other properties that are important in auction systems, such as maximality, match finality, and justified rejection, are not yet formalized in this framework. Extending the model to include these properties is a natural next step.

Another direction is to verify temporal behaviors beyond invariants. The current work focuses on properties that must hold at every point in the execution. Future work can study properties about progression over time, such as whether the protocol always reaches termination, whether phases occur in the correct order, and whether each transition step preserves the intended rules.

These properties require using temporal operators beyond invariants. For example, termination can be expressed using F (eventually), and step-by-step correctness can be expressed using X (next). These operators are already present in the LTL syntax and trace semantics of the current development; future work would add the corresponding specifications and mechanically checked proofs for MUDA traces.

Finally, the model can be used as a reference when validating real implementations of MUDA. By comparing system behavior against the formally verified model, developers can detect deviations early and build evidence that an implementation matches the verified transition system and fairness statements.
