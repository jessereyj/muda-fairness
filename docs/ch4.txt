CHAPTER 4
FORMAL LOGIC AND VERIFICATION

This chapter presents the formal language and semantics used to specify and verify properties of MUDA executions. Linear Temporal Logic (LTL) is interpreted over the infinite trace induced by the deterministic MUDA transition system from Chapter 3. MUDA-specific atomic propositions are defined as state-level predicates and are interpreted on the MUDA execution trace via a fixed interpretation function. The Rocq development mechanizes the LTL syntax and trace semantics, the MUDA-to-trace interpretation, and the proofs of the three fairness properties.

The Rocq development defines LTL with the temporal operators X (next), F (eventually), and G (always). However, the fairness properties verified in this thesis are invariance properties and are stated using G only.

The framework is organized into three layers:

Foundation layer: LTL syntax and trace semantics (see Syntax.v and Semantics.v).
MUDA protocol layer: deterministic transition system and infinite executions via terminal stuttering (see Transitions.v and Interpretation.v).
Fairness verification layer: LTL fairness formulas and mechanically checked proofs on MUDA traces (see PriorityFairness.v, QuantityFairness.v, PriceFairness.v).

4.1 Foundation Layer
Standard soundness/completeness results for LTL are cited from the literature; this development mechanizes LTL syntax and trace semantics (not a derivation/proof system), and therefore does not include mechanized soundness/completeness proofs.

The fairness properties considered in this thesis are invariance properties. Therefore, they can be expressed using the temporal operator G, and no additional temporal operators are required.

4.1.1 Syntax

Let PROP be the set of atomic propositions. In this development, PROP is implemented as ℕ (natural-number indices). Atomic propositions are written as Atom p where p : nat (see Syntax.v).

Formulas are built from atomic propositions using propositional connectives and temporal operators. The mechanization provides X, F, and G; the fairness specifications in this thesis use G only.

Propositional: ¬, ∧, ∨, →
Temporal: X (next), F (eventually), G (always)

Grammar (informal):
φ ::= Atom(p) | ¬φ | (φ ∧ ψ) | (φ ∨ ψ) | (φ → ψ) | X φ | F φ | G φ


4.1.2 Semantics

A valuation is a function v : PROP → Prop. An infinite trace is a sequence of valuations:
σ = (v₀, v₁, v₂, …)

Satisfaction σ, i ⊨ φ is defined as follows:

σ, i ⊨ Atom(p) iff vᵢ(p) holds
σ, i ⊨ ¬φ iff not (σ, i ⊨ φ)
σ, i ⊨ (φ ∧ ψ) iff (σ, i ⊨ φ) and (σ, i ⊨ ψ)
σ, i ⊨ (φ ∨ ψ) iff (σ, i ⊨ φ) or (σ, i ⊨ ψ)
σ, i ⊨ (φ → ψ) iff not (σ, i ⊨ φ) or (σ, i ⊨ ψ)
σ, i ⊨ X φ iff σ, i+1 ⊨ φ
σ, i ⊨ G φ iff for all j ≥ i, σ, j ⊨ φ
σ, i ⊨ F φ iff there exists j ≥ i such that σ, j ⊨ φ

Write σ ⊨ φ to mean σ, 0 ⊨ φ.


4.2 MUDA Protocol Layer (Executions as Infinite Traces)

A MUDA execution is generated by the deterministic transition function δ from Chapter 3. The function δ is implemented in the mechanization as the total function step : State → State (see Transitions.v).

Given an initial state x₀, the finite-step execution is defined as:
xᵢ = execute i x₀
where execute iterates step i times.

Although matching terminates, LTL requires an infinite trace. When a terminal state is reached, the execution is extended by stuttering. If phase(x) = P7, then step(x) = x. The execution therefore becomes an infinite sequence by repeating the terminal state.

Since the transition function is deterministic, each initial state induces a unique execution trace. Therefore, verification is performed over this uniquely determined trace.

4.3 Atomic Propositions and Interpretation

Atomic propositions are state-level predicates defined in Atoms.v. The mapping from PROP = ℕ to these predicates is defined in Interpretation.v using the function:

interp_atom : State → nat → Prop

The infinite trace is defined as:
mu_trace x₀, where the i-th valuation is given by interp_atom (execute i x₀).

The mapping includes:

p_allocOK, p_has_cprice, p_bounds_cstar, p_price_rule, p_prioB_step, p_prioS_step, and phase predicates.

The interpretation function correctly reflects the state-level predicates defined in Atoms.v. Therefore, satisfaction of atomic propositions corresponds directly to evaluation of these predicates on execution states.

This correspondence is formalized by the lemma mu_trace_atom_at_execute, which states that atomic propositions evaluated on the trace coincide with their evaluation on execution states.


4.3.1 Formal Priority Correctness

Priority fairness is about the correctness of the deterministic greedy selection step in Phase P3. In this thesis, that step-based notion is expressed as a predicate of the current state x by inspecting the deterministic selector computed from x.

Let x be a state. The two priority step predicates are defined in Atoms.v as follows.

Buyer-step priority correctness, `priorityB_step_ok_prop(x)`, states that if `phase(x) = P3` and the selector finds a feasible pair with chosen buyer b, then no feasible buyer with higher priority than b exists in x. Formally, for all buyers b′ in `bids(x)`, if `prioB(b′, b)` holds, then there does not exist any seller a′ in `asks(x)` such that feasibility holds for the pair (b′, a′) with respect to `matches(x)`.

Seller-step priority correctness, `priorityS_step_ok_prop(x)`, states that if `phase(x) = P3` and the selector chooses the pair (b, a), then no feasible seller with higher priority than a exists for the chosen buyer b in x. Formally, for all sellers a′ in `asks(x)`, if `prioS(a′, a)` holds and feasibility holds for (b, a′), then a contradiction follows.


4.4 Fairness Specifications (LTL)

All fairness properties are expressed as LTL formulas over the MUDA trace σ = mu_trace(initial_state(bs, as_list)).

4.4.1 Priority Fairness

Define:
priorityOK := (G (Atom p_prioB_step)) ∧ (G (Atom p_prioS_step))

This expresses that at every time index, the current state satisfies buyer-step and seller-step priority correctness, with the predicates internally guarded by Phase P3.

4.4.2 Quantity Fairness

Define:
quantityOK := G (Atom p_allocOK)

Here p_allocOK is interpreted as `allocOK_prop` in Atoms.v, which states that for each agent, the initial quantity equals the allocated quantity from the match record plus the residual quantity.

(In the mechanization, allocated quantities are computed from the match record using `allocated_bid` and `allocated_ask`, and residual quantities are derived as `residual_bid` and `residual_ask`.)

4.4.3 Uniform Price Fairness

Define:
priceOK := G (Atom p_has_cprice → (Atom p_bounds_cstar ∧ Atom p_price_rule))

This states that whenever a clearing price exists, it satisfies both boundedness by the marginal pair and correctness with respect to the deterministic pricing rule.

4.5 Fairness Verification (Theorems and Proof Sketches)

Theorems are proven for all traces generated from all initial states.

Theorem 1 (Priority Fairness).
For all inputs bs and as_list, let σ = mu_trace(initial_state(bs, as_list)). Then:
σ ⊨ (G (Atom p_prioB_step)) ∧ (G (Atom p_prioS_step)).

Mechanized theorem: `priority_fairness_LTL_initial` in PriorityFairness.v.

Proof sketch.
Fix any index j ≥ 0 and let x = execute j (initial_state(bs, as_list)). By the interpretation lemma `mu_trace_atom_at_execute`, satisfaction of atomic propositions reduces to the evaluation of the corresponding state predicates on x. Each predicate is guarded by the condition `phase(x) = P3`. When the phase condition holds, the deterministic selector ensures that no feasible order with higher priority exists. Therefore both predicates hold at every index, and their G formulas hold.

Theorem 2 (Quantity Fairness).
For all inputs bs and as_list, let σ = mu_trace(initial_state(bs, as_list)). Then:
σ ⊨ G (Atom p_allocOK).

Mechanized theorem: `quantity_fairness_LTL_initial` in QuantityFairness.v.

Proof sketch.
The bound predicate `allocOK` holds in the initial state and is preserved under each transition step. This is shown by case analysis on the phase, and induction over execute establishes the bound for all indices. A separate lemma shows that `allocOK` implies the accounting predicate `allocOK_prop` (initial quantity equals allocated quantity plus residual quantity), which is the interpretation of Atom p_allocOK. The interpretation lemma then lifts this pointwise fact to the LTL formula.


Theorem 3 (Uniform Price Fairness).
For all inputs bs and as_list, let σ = mu_trace(initial_state(bs, as_list)). Then:
σ ⊨ G (Atom p_has_cprice → (Atom p_bounds_cstar ∧ Atom p_price_rule)).

Mechanized theorem: `uniform_price_fairness_LTL_initial` in PriceFairness.v.

Proof sketch.
For each index j, satisfaction reduces to evaluation of state predicates at execute j. The pricing lemmas establish boundedness and correctness of the clearing price. The implication holds pointwise and lifts to G.


4.6 Verification vs. Satisfiability

This thesis does not search for traces satisfying a formula. It fixes the MUDA-induced trace from an initial state and proves in Rocq that this trace satisfies the fairness formulas. Since the transition system is deterministic, each initial state induces a unique infinite trace. The proofs establish that all such traces satisfy the fairness properties.
Therefore, the verification establishes that all executions of the MUDA protocol satisfy the specified fairness properties.
