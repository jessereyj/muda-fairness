Chapter 4

FORMAL LOGIC AND VERIFICATION

This chapter describes the formal language used to express and verify properties of MUDA executions.

First, it gives the syntax and trace semantics of Linear Temporal Logic (LTL) formalized in Rocq.

Second, it introduces the deterministic MUDA transition system from Chapter 3 and explains the induction of an infinite execution trace through terminal stuttering.

Third, it defines atomic propositions particular to MUDA as state predicates, and describes their interpretation on MUDA execution traces through a fixed interpretation function.

The entire machinery developed here is formalized and verified in Rocq: the LTL syntax and semantics, the MUDA trace interpretation through stuttering, and the mechanically checked proofs of the fairness properties.

This thesis uses LTL with the X, F, and G operators. Operators X and F are provided in the mechanization for completeness. However, the LTL formalization is not used for verification of fairness properties, which are invariance properties formulated using the operator G only.

All fairness properties considered in this chapter are invariant properties. Each is formulated in terms of G phi, stating that phi holds at every position along the infinite execution trace.

The framework can be viewed as comprising three layers:

foundation layer: LTL formalization in terms of syntax and semantics (see Syntax.v and Semantics.v);

MUDA protocol layer: deterministic MUDA transition system and induction of an infinite execution trace via terminal stuttering (see Transitions.v and Interpretation.v);

fairness verification layer: LTL fairness formulae and proofs over MUDA traces (see PriorityFairness.v, QuantityFairness.v, and PriceFairness.v).

4.1 Foundation Layer

This thesis employs semantic satisfaction over a fixed, MUDA-induced trace. Hence, no proof system is required. Completeness is unnecessary because the trace is used to show satisfaction rather than derivability of formulae.

As in this chapter, the properties of interest are invariance properties. They are formulated in terms of the temporal operator G.

4.1.1 Syntax

Let PROP denote the set of atomic propositions. For purposes of mechanization, atomic propositions are implemented as natural number indices. Thus, in this development PROP is implemented as nat. An atomic proposition is written as Atom p where p : nat (see Syntax.v).

Propositional logic is extended by propositional connectives. Temporal logic adds the temporal operators X, F, and G.

Propositional: ~, /\, \/, ->

Temporal: X (next), F (eventually), G (always)

The following gives the informal grammar of LTL formulae.

Grammar (informal)

phi ::= Atom(p) | ~phi | (phi /\ psi) | (phi \/ psi) | (phi -> psi) | X phi | F phi | G phi

These LTL formulae are interpreted over MUDA execution traces defined below.

4.1.2 Semantics

A valuation is a map v : PROP -> Prop. An infinite trace is an infinite sequence of valuations:

sigma = (v0, v1, v2, ...)

Satisfaction over an infinite trace is denoted by sigma, i |= phi. For a given valuation, the satisfaction predicate is defined recursively according to the following rules.

Propositional rules:

sigma, i |= Atom(p) iff vi(p)

sigma, i |= ~phi iff not(sigma, i |= phi)

sigma, i |= (phi /\ psi) iff (sigma, i |= phi) and (sigma, i |= psi)

sigma, i |= (phi \/ psi) iff (sigma, i |= phi) or (sigma, i |= psi)

sigma, i |= (phi -> psi) iff not(sigma, i |= phi) or (sigma, i |= psi)

Temporal rules:

sigma, i |= X phi iff sigma, i+1 |= phi

sigma, i |= G phi iff sigma, j |= phi for all j >= i

sigma, i |= F phi iff there exists j >= i such that sigma, j |= phi

Denote satisfaction sigma, 0 |= phi by sigma |= phi.

4.2 MUDA Protocol Layer (Executions as Infinite Traces)

This section relates the MUDA transition function defined in Chapter 3 to the trace semantics necessary for LTL satisfaction.

An MUDA execution is generated by the deterministic transition function delta of Chapter 3. The deterministic transition function is defined formally as delta : State -> State and corresponds to step in Rocq (see Transitions.v) defined as delta under thesis level alias.

Given an initial state x0, the finite execution is defined as follows.

xi = execute i x0

Here, execute i denotes application of delta i times.

Even though the transition eventually matches, LTL requires an infinite trace. Once a terminal state is reached, the MUDA execution is prolonged using stuttering. More precisely, once phase(x) = P7, then delta(x) = x. Therefore, MUDA execution becomes an infinite sequence of terminal states.

Inducing terminal stuttering ensures that every finite MUDA execution induces an infinite trace as required by LTL.

Since the MUDA transition system is deterministic, each initial state induces a unique execution trace. Hence, a proof that the MUDA trace sigma |= phi establishes that the MUDA execution trace from that state satisfies phi globally.

4.3 Atomic Propositions and Interpretation

The state-level atomic propositions are defined in Atoms.v. The mapping from PROP = nat to the state-level predicates is defined in Interpretation.v through the function interp_atom as follows.

interp_atom : State -> nat -> Prop

The function lifts state predicates into trace predicates as follows: interp_atom (execute i x0)(p) returns the truth value of atomic proposition p at position i in trace generated from state x0.

An infinite trace from state x0 is defined as:

mu(x0), whose i-th valuation is interp_atom (execute i x0)

The atomic propositions include the following predicates defined as follows:

p_allocOK, p_has_cprice, p_bounds_pstar, p_price_rule, p_prioB_step, p_prioS_step, and the phase predicates.

Each predicate corresponds directly to definitions in Chapter 3 (e.g., allocation consistency, priority selection, and clearing price correctness).

Interpretation correctly reflects state-level predicates defined in Atoms.v, so evaluation of the atom predicate on the MUDA trace corresponds directly to evaluation of the predicates on the state.

This correspondence is established in the mechanization by the following lemma.

Lemma mu_trace_atom_at_execute (forall i, atomic predicate evaluated at position i on MUDA trace coincides with that evaluated at state execute i x0).

In what follows, this lifting step will repeatedly be applied together with satisfies_always_unfold (unfolding of G in Rocq), in order to lift statements about execute i x0 predicates to LTL satisfaction over the trace mu(x0). This lifting step is referred to as LTL lift step.

The LTL lift step is used repeatedly when specifying and proving the fairness properties as LTL in Sections 4.4 and 4.5.

Both the process of stuttering and the process of LTL lifting serve different purposes.

Stuttering makes sure that every MUDA execution generates an infinite trace, which is necessary in order to make sense of an LTL formula. That is, once phase(x) = P7, it follows that delta(x) = x so that the MUDA execution becomes an infinite constant sequence. This makes sure that there is no need for special handling of terminal states in formula evaluation. Once the protocol reaches its terminal state, it is impossible to violate the fairness property further.

The LTL lifting step connects satisfaction of the LTL formulae over execution trace and satisfaction of state-level predicates. The fairness properties are formulated in terms of predicates over the MUDA execution states (e.g., allocOK(x) and bounds_pstar(x)). Yet, in the framework of LTL, formulae are evaluated over infinite traces. In essence, the LTL lift step is a step translating statements about states execute i x0 into LTL formulae over positions i of trace mu(x0) using the interpretation function.

As a result, properties formulated over all reachable states can be translated into LTL formulae, which establish the property in question for all states reachable during the execution.

4.3.1 Formal Priority Correctness

Priority fairness is concerned with the deterministic greedy selection step in Phase P3. Such a step-based notion is formalized as predicate on state x by inspecting deterministic selector.

These predicates formalize the fact that no higher-priority feasible alternative is ignored by the deterministic selector.

Suppose state x. The two predicates of priority fairness for state x are defined in Atoms.v as follows.

Definition priorityB_step_ok_prop (buyer-step priority correctness): Suppose phase(x) = P3. Assume that the deterministic selector picks a pair (b,a). Then, no buyer b' in bids(x) has higher priority compared to b and is feasible. That is, for all buyers b' in bids(x), if prioB(b', b) holds, then there does not exist any seller a' in asks(x) such that the pair (b', a') is feasible with respect to matches(x).

Definition priorityS_step_ok_prop (seller-step priority correctness): Suppose phase(x) = P3. Assume that the deterministic selector picked the pair (b,a). Then, no seller a' in asks(x) has higher priority compared to a and is feasible for the chosen buyer b in state x. That is, for all sellers a' in asks(x), if prioS(a', a) and the feasibility of (b, a') holds, then a contradiction follows.

4.4 Fairness Specifications (LTL)

Every fairness property is formulated in terms of LTL formula over trace sigma = mu(initial_state(bs, as_list)).

4.4.1 Priority Fairness

Define priorityOK := (G (Atom p_prioB_step)) /\ (G (Atom p_prioS_step))

Priority fairness says that at each moment the priority criterion is satisfied. In essence, this means that, when selecting a matching pair, the protocol should respect the priority ordering of agents. The predicate p_prioB_step and p_prioS_step guarantee that for buyers and sellers, respectively, the criteria for correct selection is met. Here, the temporal operator G is used in order to ensure that the predicates hold at every step of the protocol execution, not just during actual matching decisions. Outside Phase P3, these predicates hold trivially because matching decisions are not performed.

4.4.2 Quantity Fairness

Define quantityOK := G (Atom p_allocOK)

Quantity fairness guarantees that the total units consumed by each participant does not exceed the supply (demand) available in that participant's initial inventory. In other words, this property formalizes the conservation of units across the protocol run. The predicate p_allocOK is true if the initial quantity in state is exactly divided into allocated and residual quantity for every participant. The operator G makes sure that the allocation is always balanced.

Implementation Note (decidable equality).

For efficiency purposes, the allocation predicates have been formulated using decidable equality (see bid_eq_dec and ask_eq_dec), generated automatically using the standard Coq decide equality command.

4.4.3 Price Fairness

Define priceOK := G (Atom p_has_cprice -> (Atom p_bounds_pstar /\ Atom p_price_rule))

Price fairness specifies the fact that the uniform pricing policy of MUDA is followed. The predicate p_has_cprice is a predicate on whether the clearing price is well-defined from the match record in state x. If this is true, the clearing price must fall between the marginal buyer and seller bids, and satisfy the deterministic pricing criterion. This property is stated using implication since it only needs to hold if a clearing price is actually defined.

Note that these atoms are formulated so they are trivial when pricing is not applicable.
In particular, if no marginal pair is present (equivalently, no trades occurred and the match record is empty),
then both bounds and rule conditions hold trivially.
In addition, the deterministic price-rule atom is guarded by phases: it holds trivially in phases P1–P3,
and becomes relevant from Phase P4 onward. The bounds atom is phase-agnostic; it becomes non-trivial
exactly when the marginal pair and the computed clearing price are defined from the match record.

4.5 Fairness Verification (Theorems and Proof Sketches)

For each fairness property, a mechanized theorem is given asserting that the property holds for all traces generated from any initial state.

Theorem 1 (Priority Fairness).

For all inputs bs and as_list, let sigma = mu(initial_state(bs, as_list)). Then:

sigma |= (G (Atom p_prioB_step)) /\ (G (Atom p_prioS_step)).

This theorem states that the fairness property holds for all states of all traces induced by arbitrary input lists bs and as_list.

Rocq: priority_fairness_LTL_initial.

Proof sketch.

Fix any j >= 0 and let x = execute j (initial_state(bs, as_list)). Then, by definition, the satisfaction sigma, j |= (... G ...) is equivalent to the satisfaction of predicates on x by the interpretation lemma. Both p_prioB_step and p_prioS_step are guarded by the assumption that phase(x) = P3. In this case, the deterministic selector ensures that no better feasible orders can exist, hence both predicates are satisfied.

Theorem 2 (Quantity Fairness).

For all inputs bs and as_list, let sigma = mu(initial_state(bs, as_list)). Then:

sigma |= G (Atom p_allocOK).

This theorem states that the fairness property holds for all states of all traces induced by arbitrary input lists bs and as_list.

Rocq: quantity_fairness_LTL_initial.

Proof sketch.

The predicate allocOK holds for initial state, and is maintained by each state transition. Fix any j and consider state x = execute j (initial_state(bs, as_list)). Using induction on execute establishes the predicate allocOK(x). Lemma allocOK_to_allocOK_prop connects the allocOK(x) with atom predicate p_allocOK(x). Lifting of the predicate yields the theorem.

Theorem 3 (Price Fairness).

For all inputs bs and as_list, let sigma = mu(initial_state(bs, as_list)). Then:

sigma |= G (Atom p_has_cprice -> (Atom p_bounds_pstar /\ Atom p_price_rule)).

This theorem states that the fairness property holds for all states of all traces induced by arbitrary input lists bs and as_list.

Rocq: uniform_price_fairness_LTL_initial.

Proof sketch.

Fix any j and let x = execute j (initial_state(bs, as_list)). Then, the satisfaction sigma, j |= (...) reduces to the evaluation of pricing lemmas, which show that, in state x, if p_has_cprice(x) holds, then the clearing price is indeed bounded and follows the pricing rule.

4.6 Verification vs. Satisfiability

This thesis does not seek any traces satisfying the fairness formulae. It takes the MUDA-induced trace for arbitrary initial state and demonstrates the satisfaction of the formulae over this trace using the Rocq prover. As noted above, due to determinism of the MUDA transition system, each initial state induces an unique MUDA trace.

Hence, the verification shows that all executions of MUDA generated by transition system with inputs as initial states satisfy the fairness properties.

Thus, fairness is guaranteed for all executions generated by the MUDA transition system.
