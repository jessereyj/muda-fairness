CHAPTER 5
EXECUTION ILLUSTRATION

5.1 Execution Scenario 1

In this chapter, an example of an execution is constructed using the deterministic transition function. The example shows how the abstract state predicates and fairness conditions are checked during the execution process. It does not offer further verification, since this is done in Chapter 4 through mathematical proof.

This chapter illustrates the theorems but does not prove their correctness. It does not contain novel findings. It shows what is defined mathematically in Chapter 4.

Let sigma be an infinite execution trace, denoted by mu(x0), obtained by applying the deterministic transition function to x0.

5.1.1 Initial Orders

The buyer and seller orders are specified as follows.

Buyers:

b1 = (10, 5, 1)
b2 = (8, 4, 2)

Sellers:

s1 = (6, 3, 1)
s2 = (7, 4, 2)

Here,

The first component represents the unit price.
The second component represents the unit quantity at the initial stage.
The third component represents the submission time.

Buyer order precedence:

b1 >_B b2 since 10 > 8

Seller order precedence:

s1 >_S s2 since 6 < 7

5.1.2 Initial State

The initial state x0 satisfies the following properties.

phase(x0) = P1
residB(b1, x0) = 5
residB(b2, x0) = 4
residS(s1, x0) = 3
residS(s2, x0) = 4
M(x0) = ∅
pstar is not yet determined.

5.1.3 Execution Trace

Based on the definition of trace semantics given in Chapter 4, the deterministic execution trace starting from x0 is treated as an infinite trace. In this case, the finite execution traces are specified as follows:

xi = execute i x0.

Here, each xi is the state at position i in the trace mu(x0). The transition from execute to trace is the lifting process described in Chapter 4.

The execution trace follows the sequence of the seven protocol phases.

Note that in Phase P2, the sorting transition leads to a Phase P3 state without executing any match.

The clearing price is computed in the transition between Phases P4 and P5 and stored in the Phase P5 state.

Important alignment note (Chapter 4 atoms vs. stored field): in the mechanization, the Chapter 4 atom for
"clearing price exists" is defined via the computed function determine_clearing_price on the current match
record. As a result, the atom can become true as soon as the match record is non-empty (during Phase P3),
even though the state field clearing_price is only populated after the Phase P4 pricing step (i.e., in the
next Phase P5 state). The table below reports the stored field.

In each line, there is one state xi = execute i x0, with information about the phase, residuals, match record, and clearing price.

State | Phase | Residual Quantities | Match Record | Clearing Price
x0 | P1 | b1=5, b2=4, s1=3, s2=4 | empty | undefined
x1 | P2 | b1=5, b2=4, s1=3, s2=4 | empty | undefined
x2 | P3 | b1=5, b2=4, s1=3, s2=4 | empty | undefined
x3 | P3 | b1=2, b2=4, s1=0, s2=4 | [(b1, s1, 3)] | undefined
x4 | P3 | b1=0, b2=4, s1=0, s2=2 | [(b1, s1, 3), (b1, s2, 2)] | undefined
x5 | P3 | b1=0, b2=2, s1=0, s2=0 | [(b1, s1, 3), (b1, s2, 2), (b2, s2, 2)] | undefined
x6 | P4 | b1=0, b2=2, s1=0, s2=0 | same as x5 | undefined
x7 | P5 | b1=0, b2=2, s1=0, s2=0 | same as x5 | pstar = 8
x8 | P6 | b1=0, b2=2, s1=0, s2=0 | same as x5 | pstar = 8
x9 | P7 | b1=0, b2=2, s1=0, s2=0 | same as x5 | pstar = 8

Given that x9 is in the final phase P7, the transition function satisfies delta(x9) = x9. Thus, the trace proceeds by repeating x9, which is the stuttering process described in Chapter 4.

Following x9, the execution trace is extended by repeating x9.

5.1.4 Steps in Matching

Matching steps take place during Phase P3, corresponding to states x2 to x5 in the table.

From x2, the selector picks the feasible pair (b1, s1). The amount of trade is
q = min(residB(b1, x2), residS(s1, x2)) = min(5, 3) = 3,
yielding state x3, in which residB(b1, x3) = 2 and residS(s1, x3) = 0.

From x3, the selector picks (b1, s2). The amount of trade is
q = min(residB(b1, x3), residS(s2, x3)) = min(2, 4) = 2,
yielding state x4, in which residB(b1, x4) = 0 and residS(s2, x4) = 2.

From x4, the selector picks (b2, s2). The amount of trade is
q = min(residB(b2, x4), residS(s2, x4)) = min(4, 2) = 2,
yielding state x5, in which residB(b2, x5) = 2 and residS(s2, x5) = 0.

No feasible buyer-seller pair remains at x5, so the execution advances to Phase P4 (x6).

5.1.5 Clearing Price

The last matched pair is (b2, s2, 2).

The marginal pair determines which side sets the price.

The last seller is completely matched since residS(s2, x6) = 0. This means that the price is determined based on the buyer’s side. In this case,

pstar = p(b2) = 8

This calculation is performed during Phase P4 in the mechanization process and leads to Phase P5.

5.1.6 Priority Fairness (Illustration)

Priority fairness has been stated and proven in Chapter 4 as an invariance condition over the induced trace. In Chapter 5, no further verification is done, except to illustrate the meaning of priority atoms on this concrete run.

The meaning of priority atoms on this concrete run is as follows:

Buyer: As long as b1 is feasible, the greedy selector selects b1 before b2.
Seller: Given the chosen buyer, as long as s1 is feasible, the selector selects s1 before s2.

The above observations agree with the predicates checked at each step of Phase P3 in the trace.

It should be noted that these conditions hold at steps x2, x3, and x4.

5.1.7 Quantity Fairness (Illustration)

Quantity fairness is defined and proven in Chapter 4 as an invariance property. Scenario 1 is used to illustrate the accounting identity.

The accounting identity holds for all states of the execution trace.

In the case of b1,

initial quantity = 5
traded quantity = 3 + 2 = 5
residual quantity = 0

Then,
5 = 5 + 0

In the case of b2,

initial quantity = 4
traded quantity = 2
residual quantity = 2

So,
4 = 2 + 2

In the case of s1,

initial quantity = 3
traded quantity = 3
residual quantity = 0

Thus,
3 = 3 + 0

In the case of s2,

initial quantity = 4
traded quantity = 2 + 2 = 4
residual quantity = 0

Therefore,
4 = 4 + 0

The match record M and the residual quantities completely account for all agents’ initial quantities.

5.1.8 Price Fairness (Uniform Pricing Illustration)

Fairness conditions for price are formulated and formally proved in Chapter 4 as invariance properties linking the boundedness of prices in terms of the marginal pair with the existence of the clearing price. The following illustration shows the concrete output of the pricing mechanism in Scenario 1.

In the execution above,

pstar = 8

The result pstar, the computed clearing price, is determined from the marginal pair and the residual condition, following the deterministic pricing rule. This illustrates the output of the mechanized determine_clearing_price procedure and the subsequent storage of its value in the clearing_price field of the state.

This is a concrete example of the condition satisfied for the invariance
G (p_has_cprice -> (p_bounds_pstar /\ p_price_rule)).

In the current execution, state-level predicates, phase transitions, and deterministic procedures are shown in obtaining an execution trace that satisfies the price fairness invariants proved in Chapter 4.
